# .github/ Directory — Upstream Rebase Policy

The `.github/` directory contains files from both upstream (`vllm-project/vllm-omni`) and midstream (us). During upstream rebases, follow these rules:

## Upstream files (carry forward as-is)

These came from upstream and should be kept in sync:

- `.github/workflows/build_wheel.yml`
- `.github/workflows/pre-commit.yml`
- `.github/CODEOWNERS`
- `.github/ISSUE_TEMPLATE/`
- `.github/PULL_REQUEST_TEMPLATE.md`
- `.github/scripts/`

If upstream adds new files, carry those too.

## Midstream files (never overwrite)

Files prefixed with `midstream-` are ours. They do not exist upstream and must not be deleted or overwritten during a rebase:

- `.github/workflows/midstream-build.yml`

## How to tell them apart

The `midstream-` prefix makes ownership obvious. If you're unsure, check whether the file exists in `vllm-project/vllm-omni` — if not, it's midstream.
